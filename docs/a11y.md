# Accesibilidad y Usabilidad

Este proyecto prioriza:

- ISO 9241-11 (definición de usabilidad: eficacia, eficiencia, satisfacción)
- WCAG 2.1 nivel AA
- Diseño inclusivo, lenguaje claro y patrones accesibles

## Principios aplicados

- Perceptible: contraste ≥ 4.5:1, alternativas textuales, no solo color.
- Operable: navegación por teclado, foco visible, evitar trampas de foco.
- Comprensible: etiquetas y ayudas contextuales, validación con mensajes claros.
- Robusto: ARIA semántica, roles y propiedades correctas, HTML semántico.

## Checklist rápida (web)

- Semántica: usar `<main>`, `<header>`, `<nav>`, `<form>`, `<button>`.
- ARIA: solo cuando HTML no sea suficiente; evitar roles redundantes.
- Teclado: `Tab` lógico, `Skip to main`, `:focus-visible`.
- Contraste: paleta probada con daltonismo (herramientas simuladoras).
- Formularios: `label` asociadas, `aria-describedby` para errores.
- Lectores de pantalla: `aria-live` para resultados y errores.
- Responsivo: móvil primero, tamaños táctiles ≥ 44x44px.

## Pruebas y herramientas

- eslint-plugin-jsx-a11y, axe DevTools, Lighthouse, NVDA/JAWS, Narrator.
- Tests de teclado y lector de pantalla en vistas clave.
